# No Gamemode Change (nogamemode)

NeoForge 1.21.1 専用。このワールドでのゲームモード変更をあらゆる経路で完全に無効化する軽量Modです。

## 機能

- `/gamemode` コマンド（引数の有無・種類を問わず）を入力すると「このワールドでは無効です」と表示され、ゲームモードは変わりません。
- F3+F4 のゲームモード切替画面から選択しても変更されません（サーバー側で拒否されます）。
- LAN公開（Open to LAN）ダイアログでゲームモードを選択しても変更されません。
- サーバー起動時・ワールド参加時の初期ゲームモード割り当てはブロックしません（参加できなくなる不具合を防止）。

## 仕組み

1. `RegisterCommandsEvent` でバニラの `gamemode` コマンドノードを（リフレクションで）削除し、常に無効メッセージだけを返す代替コマンドに置き換えます。
2. NeoForge が提供する `PlayerEvent.PlayerChangeGameModeEvent`（キャンセル可能）を購読し、実際のゲームモード変更処理そのものをキャンセルします。この処理はコマンド・F3+F4・LAN公開など、ゲームモードを変更するすべての経路が最終的に通る一元的なフックのため、抜け道なく無効化できます。

## ビルド方法（おすすめ：GitHub Actionsに任せる／ローカルにJavaもGradleも不要）

1. GitHubで新規リポジトリを作成
2. リポジトリの "Add file" → "Upload files" で、このプロジェクト一式（フォルダごとドラッグ＆ドロップ可）をアップロードして commit
3. `.github/workflows/build.yml` が含まれているので、pushすると自動でビルドが走ります（リポジトリの "Actions" タブで進捗確認）
4. ビルド成功後、該当のワークフロー実行ページ下部の "Artifacts" から `nogamemode-jar` をダウンロード（zipの中に `nogamemode-1.0.0.jar` が入っています）
5. そのjarをサーバー/クライアントの `mods` フォルダに配置

ローカルにJDKもGradleも入れる必要はありません。数分待つだけです。

## ビルド方法（ローカルで行う場合）

1. Java 21 (JDK) をインストール
2. プロジェクトルートで以下を実行

```bash
./gradlew build
```

3. `build/libs/nogamemode-1.0.0.jar` が生成されるので、サーバー/クライアントの `mods` フォルダに配置してください。

## 動作確認方法

1. Modを導入したワールド（シングルプレイ／サーバーどちらでも可）を起動
2. `/gamemode creative` などを入力 → 「このワールドでは無効です」と表示され、ゲームモードは変わらないことを確認
3. チートが有効な状態で F3+F4 を押し、ゲームモードを選択 → 変更されず、チャットに「このワールドでは無効です」と表示されることを確認
4. ワールドをLAN公開する際にゲームモードを選択して開始 → 変更されないことを確認

## 注意

- 現在ゲームモードを変える手段が全くなくなるため、意図したゲームモードでワールドを作成してから導入することをおすすめします。
- サーバーの `server.properties` の `gamemode` 設定（新規参加者の初期ゲームモード）はブロック対象外です（初回割り当てのため）。
